# ADR-0001: Elpis Antigravity Edition の採用

- **ステータス**: 承認済み
- **日付**: 2026-03-12
- **決定者**: Architect

## コンテキスト

Elpis は元々 Claude Code 向けのプラグインとして設計されていた（`CLAUDE.md` + `departments/` 構成）。
Google Antigravity 環境で同等の「仮想開発組織」機能を実現するため、フレームワークの移植が必要となった。

## 決定

以下の構成変更を行い、Antigravity 環境に最適化した Edition を構築する。

1. **ルートプロンプト**: `CLAUDE.md` → `GEMINI.md`
2. **役職管理**: `departments/*/CLAUDE.md` → `.company/roles/*.md` + `index.md`
3. **ロール切替方式**: 遅延読み込み（Lazy Loading）による単一ロールファイル読込
4. **ワークフロー**: 4段階（planning → design → implement → qa_review）

## 理由

- Antigravity は `GEMINI.md` をルートプロンプトとして認識するため
- `roles/index.md` を用いたルーティングにより、Claude Code の `CLAUDE.md` チェーンと同等のコンテキスト分離を実現
- 遅延読み込みにより、コンテキストの汚染（複数ロールの混同）を防止

## 影響

- Antigravity 環境でのみ動作する（Claude Code との互換性は保持しない）
- すべてのロール定義が `.company/roles/` に集約される
- ワークフローは `.company/workflows/` で管理される
